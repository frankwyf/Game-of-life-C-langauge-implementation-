# Contributing Guide

Thanks for your interest in contributing.

## Workflow

1. Fork the repository
2. Create a feature branch
3. Make focused, minimal changes
4. Add or update tests when behavior changes
5. Open a pull request with a clear description

## Development Notes

- Main code is in `src/`
- Tests are in `test/`
- Keep changes portable across Windows/Linux where possible
- Avoid hardcoded machine-specific paths

## Coding Style

- Preserve existing C style in touched files
- Keep functions small and clear
- Add comments only when logic is non-obvious

## Pull Request Checklist

- Code builds successfully
- Tests pass locally (if environment is available)
- No personal/private data added
- Documentation updated when needed
