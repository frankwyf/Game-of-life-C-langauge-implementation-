# Security Policy

## Supported Versions

This project is a personal portfolio project. Security fixes are applied on a best-effort basis to the default branch.

## Reporting a Vulnerability

Please do not open public issues for sensitive security reports.

Use one of the following methods:

- Open a private security advisory on GitHub (preferred)
- Contact the maintainer directly through the platform profile

Please include:

- Affected file(s) and function(s)
- Reproduction steps
- Impact assessment
- Suggested mitigation (if available)

## Response Expectations

- Initial response target: within 7 days
- Fix timeline: depends on severity and project bandwidth
